# 饭桶研究所小范围灰测手册

目标：先验证“好玩 -> 晒图 -> 自愿保存档案”这条闭环，不做大规模公开推广。

## 灰测前检查

```bash
npm test
npm run check
npm run verify:supabase
```

线上状态确认：

- GitHub Actions 最新一次 Test 通过。
- GitHub Pages 状态为 `built`。
- 线上页面能看到“保存我的饭桶档案”。
- 真实邮箱能收到 6 位验证码。
- Supabase `fantong_archives` 表能看到一条测试档案。

## 灰测对象

第一轮发给 10-30 人：

- 熟人小群
- 同事/同学小群
- 对梗图、人格测试、吃饭话题比较敏感的人

暂时不要发到大流量平台，先确认验证码和海报链路稳定。

## 观察指标

手动记录这 3 个数字即可：

- 完成测试人数
- 生成报告海报人数
- 保存饭桶档案人数

建议用 `marketing/gray-test-tracker.csv` 记录。

## 反馈问题

发给灰测用户时可以问：

1. 哪一道题最像你？
2. 结果有没有想截图发群？
3. 海报保存顺不顺？
4. 邮箱验证码有没有收到？
5. 有没有哪里看起来像真实机构？如果有，具体是哪一句或哪个元素？

## 灰测判断

可以继续放量的信号：

- 10 人里至少 3 人愿意晒图或转发。
- 至少 1-3 人愿意保存档案。
- 没有人误解为真实机构。
- 手机端没有明显卡顿或布局错位。

需要先修的信号：

- 多人收不到验证码：优先配置自定义 SMTP。
- 多人不愿意保存档案：优化保存按钮利益点和同意文案。
- 多人觉得结果不够准/不够好笑：优先改题目和结果金句。
- 出现异常邮件发送量：上线 CAPTCHA 或收紧发送限制。
